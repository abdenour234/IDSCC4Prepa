Question 1
Que signifie IaaS ?
IaaS signifie Infrastructure as a Service, un modèle de cloud computing qui fournit des ressources informatiques virtualisées sur Internet.
Question 2
Quelles ressources sont typiquement incluses dans l'IaaS ?
Les ressources typiques de l’IaaS incluent le CPU, la RAM et le stockage. Les applications logicielles sont gérées par le client, pas par le fournisseur IaaS.
Question 3
Quelle est une caractéristique clé de l’IaaS ?
Le paiement à l’usage (pay-as-you-go) est une caractéristique clé de l’IaaS, permettant aux clients de payer uniquement pour les ressources utilisées.
Question 4
Dans l’IaaS, qui gère les serveurs physiques ?
Dans l’IaaS, le fournisseur Cloud gère les serveurs physiques, tandis que le client gère les systèmes d’exploitation et les applications.
Question 5
OpenStack a été initialement créé par :
OpenStack a été créé en 2010 par la NASA (projet Nebula) et Rackspace (CloudFiles).
Question 6
Quel composant OpenStack gère les machines virtuelles ?
Nova est le service de calcul (Compute Service) d’OpenStack qui gère les machines virtuelles.
Question 7
Quelles sont les fonctions de Neutron ?
Neutron gère les réseaux virtuels, incluant la création de réseaux isolés, l’attribution d’IP et le routage vers Internet. La gestion des images est assurée par Glance.
Question 8
Quel module OpenStack fournit une interface web ?
Horizon est le module d’OpenStack qui fournit une interface web pour gérer les services.
Question 9
Quel service OpenStack est utilisé pour le stockage d’objets ?
Swift est le service de stockage d’objets dans OpenStack, utilisé pour stocker des fichiers comme des backups ou des vidéos.
Question 10
Quelle commande CLI crée une image dans Glance ?
La commande glance image-create est utilisée pour créer une image dans Glance.
Question 11
Quel composant de Nova planifie le placement des VMs ?
Le composant nova-scheduler est responsable de la planification du placement des machines virtuelles.
Question 12
Quels sont les types de réseaux dans OpenStack ?
OpenStack utilise les réseaux Management, API, Data et External. Le réseau Internal n’est pas un type standard.
Question 13
Quel module OpenStack est utilisé pour l’orchestration ?
Heat est le module d’orchestration d’OpenStack, permettant de déployer des stacks d’infrastructure via des templates YAML.
Question 14
Quel service OpenStack gère l’authentification ?
Keystone est le service d’authentification et d’autorisation dans OpenStack.
Question 15
Quels sont les backends possibles pour Cinder ?
Cinder supporte des backends comme LVM, Ceph et NetApp. Swift est utilisé pour le stockage d’objets, pas pour le stockage en bloc.
Question 16
Quel composant Glance gère les métadonnées des images ?
Le composant glance-registry gère les métadonnées des images dans Glance.
Question 17
Quelle commande CLI lance une VM dans Nova ?
La commande nova boot est utilisée pour lancer une machine virtuelle dans Nova.
Question 18
Quel est le rôle de Ceilometer ?
Ceilometer est le module de télémetrie d’OpenStack, utilisé pour la supervision, la facturation et l’auto-scaling.
Question 19
Quels sont les objectifs de Ceilometer ?
Ceilometer est utilisé pour la supervision, la facturation et l’auto-scaling. L’authentification est gérée par Keystone.
Question 20
Quel module OpenStack utilise des templates YAML ?
Heat utilise des templates YAML pour déployer des stacks d’infrastructure.
Question 21
Quel composant de Neutron gère le DHCP ?
Le composant dhcp-agent de Neutron gère les services DHCP pour les réseaux virtuels.
Question 22
Qu’est-ce qu’un "project" dans Keystone ?
Un "project" (ou tenant) dans Keystone est un espace logique d’isolation pour regrouper les ressources.
Question 23
Quels sont les types d’authentification supportés par Keystone ?
Keystone supporte l'authentification par nom d'utilisateur/mot de passe, jetons et AWS-compatible. Les clés SSH sont utilisées ailleurs, comme dans Nova.
Question 24
Quel backend n’est PAS utilisé par Glance ?
Glance utilise des backends comme Swift, fichiers locaux et S3. MySQL est utilisé pour les bases de données, pas pour le stockage d’images.
Question 25
Quel composant OpenStack est exécuté sur le Controller Node ?
Keystone est exécuté sur le Controller Node, car il gère l’authentification centralisée.
Question 26
Quelle est la fonction principale de Swift ?
Swift fournit un stockage d’objets scalable, utilisé pour des fichiers comme des backups ou des vidéos.
Question 27
Quel protocole est utilisé par les services OpenStack pour communiquer ?
Les services OpenStack communiquent via des API REST.
Question 28
Quels composants sont nécessaires pour exécuter une VM ?
Pour exécuter une VM, Nova (gestion des VMs), Glance (images) et Neutron (réseaux) sont nécessaires. Horizon est facultatif car c’est l’interface web.
Question 29
Quel est le rôle du composant nova-conductor ?
nova-conductor agit comme médiateur entre la base de données et le service compute.
Question 30
Quel réseau OpenStack est utilisé pour la communication entre services ?
Le réseau Management est utilisé pour la communication entre les services OpenStack.
Question 31
Quelle est la principale différence entre Cinder et Swift ?
Cinder fournit un stockage en bloc (volumes persistants), tandis que Swift fournit un stockage d’objets (fichiers).
Question 32
Quel module OpenStack est comparable à Amazon S3 ?
Swift est comparable à Amazon S3, car il fournit un stockage d’objets via une API REST.
Question 33
Quel est le rôle de nova-api ?
nova-api reçoit les requêtes des utilisateurs pour gérer les machines virtuelles.
Question 34
Quels services utilisent RabbitMQ dans OpenStack ?
Nova, Glance et Neutron utilisent RabbitMQ comme bus de messages pour la communication interne. Horizon, étant une interface web, n’utilise pas RabbitMQ directement.
Question 35
Quel est le format de fichier typique pour les images dans Glance ?
Le format QCOW2 est couramment utilisé pour les images dans Glance en raison de ses capacités de copie sur écriture.
Question 36
Quel composant Neutron gère le routage vers Internet ?
Le composant l3-agent gère le routage vers Internet dans Neutron.
Question 37
Quel service OpenStack permet de gérer les snapshots ?
Cinder permet de créer et gérer des snapshots pour les volumes de stockage en bloc.
Question 38
Quel est le rôle du service catalog dans Keystone ?
Le service catalog dans Keystone liste les endpoints d’API des services OpenStack.
Question 39
Quels hyperviseurs sont supportés par Nova ?
Nova supporte des hyperviseurs comme KVM, QEMU et VMware. MySQL est une base de données, pas un hyperviseur.
Question 40
Quel réseau est utilisé pour exposer les APIs OpenStack ?
Le réseau API est utilisé pour exposer les APIs des services OpenStack, comme nova-api ou glance-api.
Question 41
Quel est le rôle de nova-consoleauth ?
nova-consoleauth gère l’authentification pour l’accès distant aux machines virtuelles.
Question 42
Quelle commande CLI affiche la liste des VMs ?
La commande nova list affiche la liste des machines virtuelles.
Question 43
Quel module OpenStack est utilisé pour gérer les clés SSH ?
Horizon permet de gérer les clés SSH via son interface web.
Question 44
Quels sont les avantages de Swift ?
Swift offre une haute redondance, une API REST et l’écriture via des containers. Les snapshots sont une fonctionnalité de Cinder.
Question 45
Quel composant exécute les VMs dans Nova ?
nova-compute est responsable de l’exécution des machines virtuelles dans Nova.
Question 46
Quel type de stockage est fourni par Cinder ?
Cinder fournit un stockage en bloc, utilisé pour les volumes persistants attachés aux VMs.
Question 47
Quel module OpenStack est utilisé pour la facturation ?
Ceilometer est utilisé pour collecter des statistiques d’utilisation, notamment pour la facturation.
Question 48
Quel est le rôle de neutron-server ?
neutron-server expose l’API réseau pour gérer les réseaux virtuels dans OpenStack.
Question 49
Quels services sont exécutés sur le Compute Node ?
Le Compute Node exécute nova-compute pour les VMs et neutron-agent pour les réseaux. Keystone et Horizon sont sur le Controller Node.
Question 50
Quel est le rôle principal de Horizon ?
Horizon fournit une interface web centralisée pour administrer les services OpenStack.
